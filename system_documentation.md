# GitHub 开源信息差情报站 (Open Source Info Gap Tracker)

本系统是一个基于 Node.js 构建的自动化情报雷达系统。它能够全天候监控 GitHub 开源趋势，并自动爬取全球主流社交媒体（YouTube、X、Bilibili、V2EX）的讨论热度。最终通过接入大模型（Gemini / DeepSeek），计算海内外的信息差指数，为您筛选出最具商业变现（如小红书引流、代部署服务等）潜力的优质项目。

---

## 🛠️ 技术文档 (Technical Documentation)

### 1. 系统架构
本应用采用前后端分离的轻量化架构设计：
*   **前端引擎**：基于原生 HTML/JS/CSS，结合 Chart.js 进行数据可视化，界面采用现代毛玻璃与动态 UI 设计。
*   **后端引擎**：Node.js + Express 服务。
*   **数据存储**：使用 `lowdb` / SQLite (以 `database.js` 为核心) 进行持久化存储，利用 `ON CONFLICT` 防重叠设计避免无用数据堆积。
*   **自动化调度**：内置基于 `node-cron` 或定时器的 `scheduler.js`，保证情报库持续自我刷新。

### 2. 核心模块与文件结构
*   **`src/app.js`**: 后端 API 入口，负责提供前端界面的数据接口与静态路由。
*   **`src/fetch/githubTrending.js`**: 负责对接 GitHub API，拉取 Top 20 爆发期开源仓库。
*   **`src/fetch/socialTrending.js`**: 并发抓取四方社媒热点。
    *   **海外源**：YouTube (通过动态计算近 30 天的高曝光量视频)；X (Twitter) 热点。
    *   **国内源**：Bilibili (过滤超 90 天的陈旧内容，保留近期爆款)；V2EX (IT 技术圈当日热搜)。
*   **`src/classify/aiAnalyst.js`**: 整个系统的“大脑”。负责将原始冰冷的数据打包成语义化 Prompt，交由 LLM 进行交叉对比。支持 Gemini 和 DeepSeek 的主备故障转移 (Failover) 机制。
*   **`src/push/feishuLedgerSender.js`**: 飞书多维表台账同步服务。基于本地 `lark-cli` 运行，用于将生成的自媒体运营文案推送到指定的飞书多维表（包含自动提取标题和平台映射）。


### 3. 核心计算逻辑：信息差打分模型
本系统最具价值的“热度对比与信息差打分”，主要依赖于大模型结合固定数学公式完成：

AI 大模型根据输入的数据池为每一个项目在三个维度打分 (满分 100 分)：
1.  **`github_score` (代码热度)**：依据项目的 Stars 总量和今日新增速度得出。
2.  **`social_score` (海外社媒热度)**：依据 YouTube 和 X 上的视频播放量、推文曝光量得出。
3.  **`domestic_score` (国内热度)**：依据 Bilibili 播放量和 V2EX 讨论度得出。

**🔥 信息差指数（Info Gap）计算公式：**
> **`info_gap = (github_score + social_score) / 2 - domestic_score`**

*   **计算含义**：将（代码技术圈的热度 + 海外极客玩家的热度）取平均值，减去（国内普通互联网用户的认知热度）。
*   **商业价值**：计算结果的分值越大（例如 +45），意味着这个神器在国外已经火遍大江南北，但是国内互联网（如 B站、小红书）却几乎搜不到教程。这就是绝佳的**“时间差、信息差红利”**。系统会直接将此类项目标记为 **【强推】**。

### 4. 智能文案生成引擎与自媒体矩阵接口
本系统配备了针对副业引流与内容分发的**智能文案生成引擎**，在 `src/classify/aiAnalyst.js` 中集成了多维度文案适配逻辑：
*   **API 路由接口**:
    *   `POST /api/repositories/:id/generate-copy`: 触发小红书 (XHS)、自媒体周报文章段落 或 60秒短视频口播脚本的千人千面动态文案生成。
    *   `POST /api/operations/generate`: 支持个性化特殊要求 (Custom Prompt) 和自定义引流动作 (CTA) 的单项目“一鱼两吃”智能文案并行生成。
    *   `POST /api/operations/weekly`: 支持将选中的多个项目聚合生成一篇微信公众号技术合集干货周报。
    *   `POST /api/xhs/search` & `/api/wechat/search`: 调用红狐数据 API 检索小红书/公众号真实爆款赛道与 10W+ 榜单数据。
    *   `POST /api/feishu/sync-ledger`: 一键同步当前生成的文案数据至飞书多维表台账。
*   **飞书智能同步与标题提取机制**:
    *   **本地 lark-cli 桥接**: 系统利用后端执行 `lark-cli api POST` 底层通用指令，绕过 Wiki 关联多维表对 `baseToken` 的严格正则过滤，将文案写入多维表。这需要本地提前安装并登录飞书 CLI (`lark-cli login`)。
    *   **标题智能提取启发算法**: 在向飞书推送数据时，系统并不直接把大文本塞入标题格，而是使用正则与启发式算法智能提取标题：
        1. 优先提取文案中的 Markdown 一级/二级标题行 (`#` 或 `##`)。
        2. 其次匹配以 Emoji 或中文开头、字数小于 80 字符的短行（通常为小红书等自媒体的吸睛爆款标题）。
        3. 若都不满足，则兜底截取第一个非空行的前 100 字符。
    *   **字段对齐与映射**:
        *   `平台`: 自动根据当前所选平台在飞书的多维表单选字段中映射为 `"小红书"` 或 `"公众号"`。
        *   `内容标题`: 填入上述算法智能提取出的短标题。
        *   `发布状态`: 默认填充为 `"草稿"`。
        *   `数据记录时间` & `发布时间`: 记录为推送时的时间戳。
        *   `备注`: 写入完整的 AI 生成 Markdown 文案，方便后续微调及一键复制。

*   **大模型自媒体适配机制**:
    *   **千人千面动态生成**: 摒弃了传统的固定格式模板。系统会通过自动拉取并阅读 GitHub 项目的说明书 (README)，让大模型深入分析其核心功能、痛点解决场景以及**具体的目标使用人群群体**。
    *   **自适应平台调性**: 基于分析出的人群画像，大模型会动态生成千人千面、毫不千篇一律的定制化文案。无论是小红书种草文案、微信长图文，还是短视频口播脚本，都会根据项目的实际调性和受众自动调整行文风格与切入角度，确保每篇内容都具有高度独创性和网感。

## 📖 使用说明 (User Manual)

### 1. 启动服务
在项目根目录打开终端，输入以下命令启动系统：
```bash
npm start
# 或使用 node 启动
node src/app.js
```
启动成功后，在浏览器访问：`http://localhost:3000`

### 2. 仪表盘看板导航
*   **数据大盘统计**：页面最顶端会显示上一次更新的时间、当前数据库收录的项目总数，并提供“立即抓取 (Fetch Now)” 按钮随时强制刷新最新数据。
*   **综合热度评估报告 (Top Picks)**：
    *   展示经过 AI 层层过滤筛选出的 Top 10 重点项目。
    *   带有 **[强推]** 徽章的项目代表其 **信息差极高**，强烈建议马上操作发文。
    *   点击 **[📋 复制文案]** 按钮，一秒钟即可将内置了网感 Emoji 的“小红书爆款文案”存入剪贴板，直接粘贴发布。
    *   点击 **[🚀 发文排版]** 可打开一键排版工作室。
*   **双轴对比图 (Social Compare Chart)**：
    *   直观对比这 10 个项目在 GitHub、海外社媒、国内社媒的三维分数差异。绿色柱子远高于红色柱子的，即为重点。
*   **全量开源仓库表 (Repository Grid)**：
    *   罗列出本次抓取的所有底层开源仓库数据。带有基础数据（Language, Stars, Forks）和 AI 的一句话概括。

### 3. 🚀 爆款自媒体引流文案生成
系统内置了专门面向搞钱自媒体与副业运营的**“引流文案生成引擎”**，您可以通过以下几种方式使用：

*   **一键复制标准小红书文案**:
    *   在主页面的 Top Picks 项目列表或 Repository Grid 中，点击 **[📋 复制文案]** 按钮，即可调用大模型快速渲染出符合小红书风格的引流种草文案。
*   **发文排版工作室 (智能矩阵运营文案)**:
    *   在自媒体运营中心，可对单个项目智能定制文案，核心特性支持：
        *   **智能受众定位**: 系统会自动阅读项目 README，智能分析最契合的目标人群（如职场打工人、副业淘金者等），精准击中痛点。
        *   **自动风格适配**: 根据定位出的人群画像，大模型自动匹配文案语气风格，摒弃僵化的固定套路。
        *   **个性化要求与引流**: 支持输入额外的特殊要求 (Custom Prompt) 和自定义的引流引导动作 (CTA)。
        *   **一鱼两吃生成**: 系统会自动并行生成 **小红书“诱饵”图文** 和 **微信公众号“干货包”教程**，实现双平台协同引流与粉丝沉淀。
*   **一键获取生图提示词分镜矩阵 (Prompt Matrix)**:
    *   **分平台图解矩阵**: 针对当前流行的信息流图解风格，一键提取出由 6 个不同分镜头组成的“神级提示词”。
    *   **小红书 (3张)**: 锁定 3:4 比例，涵盖“夸张大标题封面”、“痛点核心拆解”、“成果收益数据对比”。
    *   **微信公众号 (3张)**: 锁定 16:9 比例，涵盖“横屏高级宏大首图”、“科技连线网络原理图”、“未来感行动愿景图”。
    *   **高密度强指令**: 强制大模型为每张图注入“高密度、包含大量排版文字信息 (Typography-heavy, text-rich)”的强视觉约束指令，直接粘贴到 Gemini Pro 或 Midjourney 即可生成包含文字的高级海报图。
*   **多项目聚合微信周报**:
    *   勾选多个高潜力项目，点击生成周报。系统会自动撰写 3 个微信推文大标题、前沿大局行业视点、各项目商业盘点、Markdown 横向对比分析表，以及落地实战避坑指南，完美便于一键粘贴到微信公众号后台发布。
*   **一键同步飞书内容台账**:
    *   在文案排版工作室生成小红书或微信文案后，直接点击底部的 **[☁️ 同步到台账]** 按钮。
    *   系统会自动调用飞书 CLI 接口进行同步，将生成的数据自动录入至您的飞书多维表。同步成功后会弹出“成功同步至飞书台账”提示，免去在两端来回复制粘贴的琐碎工作。


### 4. 🔍 爆款赛道探测器 (Social Media Detector)
系统集成了红狐数据 API，支持穿透主流社媒平台数据壁垒，实时检索和探测高转化爆款数据：
*   **🔴 小红书爆款赛道探测**: 支持根据关键词搜索小红书真实爆款赛道数据，实时拉取全网大盘高互动量笔记，为开源变现定位精准对标账号和爆款笔记模板。
*   **🟢 微信公众号爆款搜索与 10W+ 榜单**: 支持根据关键词搜索微信公众号近期的爆款技术/搞钱文章；同时支持一键获取公众号在不同细分赛道的 10W+ 爆款文章排行榜，助力精准选题与网感培养。

### 5. 小贴士与隐藏功能
*   **自动防封锁机制**：从工具跳转 B站 会自带反热链 `rel="noreferrer"` 属性，告别 B站烦人的 “412 安全风控阻断”。
*   **内容新鲜度控制**：AI 抓取的 YouTube 内容已经被强制限定在 **"本月最新"**；B 站内容则被锁死在 **"最近 90 天内"**；您看到的情报永远是这个月热气腾腾的刚需。
